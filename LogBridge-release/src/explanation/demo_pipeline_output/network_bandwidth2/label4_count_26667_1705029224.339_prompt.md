## Tuple 2: `ISS-8403` (sim=0.846)

- Title: Cluster insert with batchSize=10000 throws `RatisRequestFailedException: Ratis request failed`
- Symptom: Insert Tablet with rowCount=10000 → `605: RatisRequestFailedException`; same code with rowCount=1000 succeeds; problem appears near completion of 3.77M-row batch insert
- Root cause: Ratis consensus restricts single write request to ≤4MB; batchSize=10000 with 30 fields × 100 measurements produces ~6MB per request, exceeding limit
- Fix:
      Raise data_region_ratis_log_appender_buffer_size_max to 16777216 (16MB) in iotdb-common.properties on ConfigNode
      Restart cluster
      Or reduce client batch size to keep request <4MB
